# DeathManager
![Version](https://img.shields.io/badge/version-1.0-blue.svg)
![API](https://img.shields.io/badge/API-1.21.x-green.svg)

**DeathManager** — легковесный и полностью настраиваемый плагин-ядро для управления смертью игроков на вашем Minecraft-сервере. Отличный выбор для создания мини-игр или кастомных сборок!
## Особенности
* ⚡ **Отсутствие экрана смерти**: Мгновенно восстанавливает здоровье, позволяет избежать стандартного экрана "Вы погибли". Способен переводить игрока сразу в режим Spectator (наблюдатель).
* 🩸 **Эффекты при смерти**: Удары молнией, эффекты крови при получении урона и смерти.
* 📦 **Настраиваемый дроп**: Детальная настройка шанса (в %) выпадения вещей и опыта при смерти. 
* 👻 **Голограммы**: Кастомные голограммы на месте гибели игрока с именами жертвы и убийцы.
* ⏳ **Умный авто-респавн**: Экранные тайтлы-таймеры, отсчитывающие время до возрождения.
* 🎯 **Авто-таргетинг**: Камера убитого игрока автоматически прикрепляется к его убийце (режим наблюдателя).
* ❤️ **Хардкорные сердца**: (Опционально) Использование `ProtocolLib` для визуальной замены стандартных сердечек на "хардкорные".

## Конфигурация (config.yml)
В конфигурационном файле есть комментарии, которые помогут вам легко настроить плагин под любую мини-игру. Можно включить и отключить практически любую фишку ядра!
## Для разработчиков (API)
Так как это плагин-сервис для остальных ваших плагинов на мини-игры, вы можете вызывать баккитовский `PlayerDeathEvent` для детекта любых смертей так же, как всегда. Плагин не ломает другие ивенты!
Вы также можете менять поведение самого менеджера на лету:
```java
// Меняем шанс дропа в коде
DeathManager.getPlugin(DeathManager.class).getSettings().dropPercentage = 50;
```
